# Успешный кейс: `test-weird-dupes` (песочница, дедуп, русские папки)

**Дата:** 2026-04-18  
**Запросы пользователя (цепочка):** применить `file-cleaner` к песочнице с кучей форматов и дубликатами; затем переименовать папки на русский язык.

## Почему кейс считается эталонным

1. **Структура из смысла** — не копировали `_КОНТЕКСТ/брифы` из других агентов. По ролям файлов выделили пять кластеров: настройки, код, данные, демо/разметка, журналы.
2. **Дедупликация** — оставили один канонический файл на группу совпадающего содержимого; `.bak` и «копии» удалили; пустой `subfolder/` убрали.
3. **Документация в зоне работы** — в корне песочницы обновили `README.md` и добавили отчёт `2026-04-18_file-cleaner.md`.
4. **Русские имена папок** — по явной просьбе пользователя переименовали каталоги; содержимое файлов не ломали.

## Использованная схема папок (русские имена)

| Роль кластера | Папка | Что класть |
|----------------|-------|------------|
| Параметры и форматы настроек | `конфигурация/` | json, toml, yaml, ini, env |
| Исполняемые сниппеты | `код/` | py, js и т.п. |
| Фикстуры и табличные/текстовые данные | `данные/` | csv, txt, md-заметки |
| Разметка и нетипичные демо | `примеры/` | html, xml, «странные» расширения |
| Журналы | `логи/` | log |

До русификации папки назывались `config`, `code`, `data`, `samples`, `logs` — смысл тот же; переименование было механическим.

## Что повторять в следующих задачах

- Сначала **прочитать смысл** файлов в области, предложить кластеры и имена (на языке, который просит пользователь или как уже принято в проекте).
- **Дедуп** по содержимому до/после раскладки — один файл на группу, каноническое имя.
- После уборки — **обновить README** в зоне (и при необходимости короткий отчёт с датой).
- Если пользователь просит **русские имена папок** для такого же типа задачи (смешанные технические файлы без чужой иерархии), **можно опираться на таблицу выше** как стартовую точку, адаптируя под контент (не добавлять лишние уровни).

## Как воспроизвести у себя

Создай в workspace **отдельную папку-песочницу** (имя на твоё усмотрение), наполни её смешанными файлами и дубликатами, запусти уборщика. Итог удобно фиксировать в корне песочницы: обновлённый `README.md` и отчёт вида `ГГГГ-ММ-ДД_file-cleaner.md` (как в п. 3 выше).
